# 第二部分：心理学与心智模型

## 基础思维模型

### 第一性原理
将问题分解为基本真理。不要复制竞争对手——反复问"为什么"以找到根本原因。

**营销应用**：不要因为竞争对手做内容营销你就做。问为什么，它解决什么问题，是否有更好的解决方案。

### 待完成工作（JTBD）
人们"雇佣"产品来完成一项工作。关注结果，而不是功能。

**营销应用**：买电钻的人想要的是洞，而不是电钻。围绕完成的工作来构建框架。

### 能力圈
知道你擅长什么并保持在其中。在真正的专业知识上加倍下注。

**营销应用**：不要追逐每个渠道。专注于你有真正竞争优势的地方。

### 逆向思维
问什么会保证失败，然后避免那些事情。

**营销应用**：列出所有会让活动失败的事情，然后系统地防止每一个。

### 奥卡姆剃刀
更简单的解释通常是正确的。避免过度复杂化战略。

**营销应用**：如果转化率下降，先检查明显的（表单损坏、页面慢），然后再进行复杂的归因。

### 帕累托法则（80/20）
80% 的结果来自 20% 的努力。找到并专注于关键的少数。

**营销应用**：找到驱动最多结果的渠道。削减或减少其余的。

### 希克定律
决策时间随选项增加。更多选择 = 更多放弃。

**营销应用**：一个明确的 CTA 胜过三个。更少的表单字段 = 更高的转化。

### AIDA 漏斗
注意 → 兴趣 → 欲望 → 行动

**营销应用**：构建页面以通过每个阶段。在建立欲望之前先抓住注意。

### 边际收益递减定律
在某一点之后，额外投资产生的收益逐渐减少。

**营销应用**：第 10 篇博客文章不会有第一篇那样的影响。多样化渠道。

### 承诺与一致性
一旦人们承诺某事，他们就想保持一致。

**营销应用**：先做小承诺（邮件注册）会导致更大的承诺（付费订阅）。

### 互惠原则
先给予。人们感到有义务回报恩惠。

**营销应用**：免费内容、工具和免费增值模型创造互惠义务。

### 稀缺性与紧迫感
有限的可获得性增加感知价值。

**营销应用**：限时优惠、低库存警告。仅在真实时使用。

### 损失厌恶
损失的感觉比同等收益的感觉好两倍痛苦。

**营销应用**："不要错过"胜过"你可以获得"。用他们会失去什么来构建框架。

### 锚定效应
第一个数字严重影响后续判断。

**营销应用**：先显示更高的价格（原价、竞争对手、企业）以锚定期望。

### 选择悖论
太多选项让人不知所措。更少的选择导致更多决策。

**营销应用**：三个定价层级。推荐一个"最适合大多数人"的选项。

### 禀赋效应
人们一旦拥有某物就更加珍视它。

**营销应用**：免费试用、样品、免费增值模型让客户"拥有"产品。

### 宜家效应
人们更加珍视他们付出努力创造的东西。

**营销应用**：让客户自定义、配置、构建。他们的投入增加了承诺。

### 单纯曝光效应
熟悉产生好感。持续的存在建立偏好。

**营销应用**：跨渠道重复创造舒适和信任。

### 社会证明 / 从众效应
人们跟随其他人正在做的事情。流行度信号质量。

**营销应用**：显示客户数量、推荐、"热门"指标。

### 前景理论 / 损失厌恶
人们避免可能导致后悔的行动。

**营销应用**：退款保证、免费试用减少后悔恐惧。直接解决担忧。

### 蔡格尼克效应
未完成的任务占据心智。开放式循环制造紧张感。

**营销应用**："你已完成 80%"创造完成的拉力。不完整的个人资料、放弃的购物车。

### 现状偏见
人们偏好当前状态。改变感觉有风险。

**营销应用**：减少摩擦。让过渡感觉安全。"一键导入"。

### 默认效应
人们接受预先选择的选项。默认值很强大。

**营销应用**：预先选择你想让客户选择的计划。选择退出胜过选择加入。

### 峰终定律
人们通过高峰（最好/最差）和结束来判断体验，而不是平均值。

**营销应用**：设计难忘的高峰和强烈的结尾。感谢页面很重要。

# 第三部分：SEO 与内容

## SEO 审计框架

### 优先级顺序
1. **可爬取性与索引** - Google 能找到并索引页面吗？
2. **技术基础** - 网站快速且功能正常吗？
3. **页面优化** - 内容是否优化？
4. **内容质量** - 它值得排名吗？
5. **权威性与链接** - 它有可信度吗？

### 技术 SEO 检查清单

**可爬取性**
- Robots.txt 没有阻止重要页面
- XML 站点地图可访问且已更新
- 站点架构在首页 3 次点击内
- 没有孤立页面

**索引**
- 重要页面没有意外的 noindex
- 正确的规范标签（自引用）
- 没有重定向链
- 没有软 404

**核心 Web 指标**
- LCP < 2.5s
- INP < 200ms
- CLS < 0.1
- 服务器响应时间优化
- 图片优化

**页面**
- 标题标签优化（60 字符，关键词位置）
- 元描述引人入胜（155 字符）
- 标题层次结构（H1 → H2 → H3）
- 内部链接到优先页面

**E-E-A-T**
- 展示作者专业知识
- 明确的来源和引用
- 定期内容更新
- 准确、全面的信息

## 要问的问题（营销发现）

**关于产品与受众**
- 你的产品是什么，目标客户是谁？
- 你目前的阶段是什么（发布前 → 规模化）？
- 你的主要营销目标是什么？
- 你的预算和团队规模是多少？

**关于当前状态**
- 你尝试过什么，有效或无效？
- 你的竞争对手在哪些方面做得好？
- 你在漏斗的哪里失去客户？

**关于目标**
- 什么指标最重要（流量、线索、收入）？
- 你的时间表是什么？
- 你的竞争优势是什么？

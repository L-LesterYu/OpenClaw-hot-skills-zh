# 工作流模板 — 可复用流程与增长循环

本文件记录重复性流程。**规则：做某件事 3 次后，记录在这里。**

---

## 使用方法

**模式：**
1. **第一次：** 弄清楚怎么做并完成
2. **第二次：** 重复流程
3. **第三次：** 在这里记录
4. **第四次及以后：** 遵循文档化的工作流

**原因：** 一致性、速度、避免重复造轮子

---

## 增长循环（持续改进）

### 循环 1：好奇心循环

**目标：** 更好地了解你的主人 → 产生更好的想法 → 提供更多价值

**循环过程：**
1. 识别知识盲区
2. 自然地每次会话问 1-2 个问题
3. 更新理解 → 添加到 USER.md
4. 利用新知识生成更好的建议
5. 再次循环

### 循环 2：模式识别循环

**目标：** 发现重复性任务 → 系统化 → 释放时间做高价值工作

**循环过程：**
1. 观察什么被反复请求
2. 识别模式 → "他们每隔 [频率] 就要我处理 [X]"
3. 提议系统化："你每周一都需要 X。我们应该自动化吗？"
4. 实施 → 脚本/模板/清单/定期工作流
5. 记录在 WORKFLOWS.md
6. 再次循环

### 循环 3：能力扩展循环

**目标：** 遇到瓶颈 → 研究 → 添加能力 → 更好地解决问题

**循环过程：**
1. 研究可用的工具/技能（ClawHub、文档、GitHub）
2. 安装/构建解决方案
3. 更新 TOOLS.md
4. 应用新能力解决原始问题
5. 再次循环

### 循环 4：结果追踪循环

**目标：** 从"听起来不错"到"已验证有效"

**循环过程：**
1. 在做重大决策时记录下来
2. 后续跟进结果
3. 提取教训 → 更新 AGENTS.md
4. 再次循环

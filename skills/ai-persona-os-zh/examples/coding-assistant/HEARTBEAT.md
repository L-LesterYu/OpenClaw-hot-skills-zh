# 心跳检查清单

## 迁移检查（运行一次）
- 如果此文件超过 50 行，则为过时模板。请从 assets/HEARTBEAT-template.md 处的当前技能模板更新您的工作区副本，然后重新运行心跳。

## 版本检查
- 读取工作区中的 VERSION.md。如缺失：从技能 assets/VERSION.md 复制。
- 比较工作区 VERSION 与技能版本（1.4.1）。如不同：标记需要升级。

## 上下文保护
- 检查上下文百分比。如 ≥70%：立即将检查点写入 memory/YYYY-MM-DD.md。跳过其余所有检查。
- 如上次检查点超过 30 分钟且上下文 >50%：在继续前写入检查点。

## 记忆维护
- MEMORY.md 是否存在？如缺失：从最新检查点或会话笔记创建。
- MEMORY.md 大小？如 >4KB：将 30 天前的条目归档到 memory/archive/memory-overflow-YYYY-MM-DD.md。仅保留活跃的、当前的事实，并报告已归档的内容。
- memory/ 中有过期日志吗？如有超过 90 天的：移至 memory/archive/。
- 昨天日志中有未完成项目吗？展示它们。

## 开发检查
- 有运行中的后台任务（构建、测试、部署）吗？检查状态。
- 有等待审查的开放 PR 吗？如超过 24 小时则标记。
- CI/CD：如可访问则检查最近管道状态。

## 报告格式（严格）
第一行必须为：🫀 [当前日期/时间] | [您的模型名称] | AI Persona OS v[VERSION]

然后每个指标必须独占一行，指标之间留空行：

🟢 上下文：[%] — [状态]

🟢 记忆：[同步状态 + 大小]

🟢 工作区：[状态]

🟢 任务：[状态]

🟢 CI/CD：[状态]

根据需要将 🟢 替换为 🟡（需关注）或 🔴（需操作）。
如有操作执行：添加以 → 开头的行描述已执行的操作。
如有需要用户关注的事项：添加以 → 开头的行并说明具体情况。
如检测到版本不匹配：添加 → 升级可用：工作区 v[旧版本] → 技能 v[新版本]
如所有指标均为 🟢、无操作执行且无可用升级：仅回复 HEARTBEAT_OK
不要使用 markdown 表格。不要使用 Step 0/1/2/3/4 格式。不要使用标题。

# 公文写作 Skill

> 基于2026年3月17日公文比对分析报告的改进成果，本文档是 DW Agent 撰写政府公文的必备指南。

---

## 一、写作流程（必须按顺序执行）

```
第1步：理解需求
  └─ 确认公文类型、目标读者、目的、主题、是否本地化

第2步：本地化信息搜集（如涉及具体城市）
  └─ 搜索目标城市：最新政府工作报告、重点产业政策、人才工程、
     产业园区/平台、地方立法进展
  └─ 搜集结果存为临时参考，确保建议中引用3+个本地资源

第3步：资料搜索与事实核查
  └─ 涉及人物身份：至少2个独立来源交叉验证
  └─ 涉及政策日期：必须抓取原文确认
  └─ 涉及监管/风险：搜索最近3个月官方通知
  └─ 涉及补贴数据：获取具体金额、比例、条件

第4步：起草正文
  └─ 严格按结构模板撰写
  └─ 每条建议检查"四要素"齐全
  └─ 风险分析按"四段式"展开

第5步：附件准备
  └─ 判断是否需要附件
  └─ 需要则搜集原始文件全文

第6步：自检（按下方检查清单逐项核对）
```

---

## 二、公文结构模板

```
标题（关于XXX的有关情况及建议）

[引言段：背景交代 + 战略意义 + 本文目的，2-4句话，至少引用一项上级文件]

一、[一级标题]
[正文内容]
（一）[二级标题]
[正文内容]
1. [三级标题]
[正文内容]

[注意：禁止自主添加落款，除非用户明确要求]
```

### 结构要求

- **引言段必写**：交代宏观背景，点明战略意义，说明本文目的
- **建议部分**：每条建议下设"一是、二是、三是"子项，形成完整链条
- **逻辑递进**：采用"引言—现状/背景—问题/风险—建议"链条

## 三、编号体系规范

| 层级 | 格式 | 示例 |
|------|------|------|
| 第一层 | 中文数字 + 顿号 | 一、二、三、四 |
| 第二层 | 括号中文数字 | （一）（二）（三） |
| 第三层 | 阿拉伯数字 + 点号 + 空格 | 1. 2. 3. |
| 第四层 | 括号阿拉伯数字 | （1）（2）（3） |

**核心规则：全文只允许使用一套编号体系，禁止混用。**

## 四、事实核查清单（P0 级，必须执行）

### 人物身份核查
- [ ] 姓名拼写正确（中英文对照）
- [ ] 职务/身份有可靠来源
- [ ] 履历信息经至少2个来源验证
- [ ] 网名/昵称与真实姓名的关系已确认

### 政策文件核查
- [ ] 发布日期已通过原文确认（精确到日）
- [ ] 发布机构名称准确
- [ ] 文件名称/文号正确
- [ ] 政策内容引用准确，非二手转述

### 数据核查
- [ ] 数据有具体来源
- [ ] 补贴金额/比例/条件完整
- [ ] 同一数据在不同来源中一致
- [ ] 日期格式统一（XXXX年X月X日）

### 禁止事项（红色警告）
- 🚫 **禁止自主添加落款**（单位名称 + 日期），除非用户明确要求
- 🚫 **禁止编造人名、日期、数据**
- 🚫 **禁止将推测写成确定事实**

## 五、内容深度要求

### 建议部分"四要素"标准

每条建议必须包含：

| 要素 | 说明 | 自检问题 |
|------|------|----------|
| 主体 | 谁来做 | "由哪个部门/机构牵头？" |
| 动作 | 做什么 | "具体措施是什么？" |
| 目标 | 做到什么程度 | "预期成效/量化指标？" |
| 保障 | 用什么资源 | "资金/政策/人才保障？" |

### 建议标题格式

使用**四字动词短语**：
- 明确发展定位
- 出台专项政策
- 建设基础设施
- 培育应用生态
- 加强人才培养
- 完善安全机制
- 营造发展氛围
- 建立协调机制
- 强化政策供给
- 优化营商环境

### 建议子项结构

```
（X）[四字动词短语]。[宏观定位 + 具体措施 + 政策依据 + 实施路径]。

一是[子项1]。[具体措施，引用政策/数据]。
二是[子项2]。[具体措施，引用政策/数据]。
三是[子项3]。[具体措施，引用政策/数据]。
```

### 风险分析"四段式"模板

每类风险按以下结构展开：
1. **风险表现**：具体是什么风险
2. **影响范围**：影响哪些群体/行业
3. **危害程度**：可能的后果（区分个人用户 vs 企业 vs 关键行业）
4. **防护建议**：具体可操作的防护措施

## 六、本地化适配要求

### 写作前必须搜集的信息

1. **城市定位**：官方表述（如"全国先进制造业基地和单项冠军之城"）
2. **政府工作报告**：最新年度报告中与本主题相关的部署
3. **重点政策**：地方立法、专项政策、发展规划
4. **人才工程**：地方人才引进计划名称和具体内容
5. **产业平台**：产业园区、科创区、数字港等载体名称
6. **政策风格**：当地常用的政策表述和机制名称

### 本地化检验标准

- [ ] 开篇点明城市定位
- [ ] 建议中引用至少3个本地具体资源（政策名称、平台名称、人才工程等）
- [ ] 建议"换城测试"：将城市名替换为其他城市后，至少有50%内容不适用
- [ ] 建议措辞贴合当地政策风格

## 七、附件要求

### 需要附件的公文类型

| 公文类型 | 必需附件 |
|----------|----------|
| 政策梳理类 | 相关政策文件全文 |
| 风险提示类 | 官方风险提示/通知全文 |
| 调研报告类 | 调研数据、问卷、统计表 |
| 对策建议类 | 政策依据文件、对比分析数据 |

### 附件格式规范

- 附件标题格式：`附件1：[附件名称]`
- 内容为完整原文，非摘要
- 注明来源和获取日期
- 正文中引用处标注"详见附件X"

## 八、语言表达规范

### 公文常用表达

**开篇引入：**
- "根据……要求，现将……情况汇报如下"
- "为贯彻落实……精神，推动……发展"
- "近期，……（背景），现将有关情况及建议报告如下"

**过渡衔接：**
- "基于上述分析，提出以下建议"
- "结合……实际，建议从以下几个方面着手"
- "借鉴……经验，针对……问题"

**建议表述：**
- "建议将……作为……的重要切入点"
- "建议借鉴……经验，研究出台……"
- "建议与……合作，建立……机制"
- "建议成立由……牵头、……参与的协调小组"

**数据引用：**
- "截至目前，……已达……"
- "据……统计，……"
- "于……年……月出台……"

### 修辞手法（每篇至少2处）

| 类型 | 用法 | 示例 |
|------|------|------|
| 形象类比 | 用熟悉事物解释新概念 | "咨询顾问"vs"数字管家" |
| 排比对仗 | 三字/四字短语并列 | "选准试验田、培育企业群、做优服务链" |
| 对比修辞 | 新旧对比、正反对比 | "彻底打破能说不能做的局限" |
| 提炼金句 | 可独立引用的核心观点 | "掀起一场从对话到行动的范式革命" |

### 禁用词/表达列表

| 禁用 | 替换为 |
|------|--------|
| 零成本启动 | 降低准入门槛 |
| 全链条赋能 | 提供全流程服务保障 |
| 被亲切地称为 | （直接删除） |
| 7×24小时 | 全天候 |
| 咨询电话被打爆 | 咨询量显著增长，反映市场关注度高 |
| 线下沙龙座无虚席 | 各类线下活动参与度较高 |
| 我们 | （避免第一人称，使用客观表述） |
| 我 | （避免第一人称） |

### 中英文混排规范

- 中英文之间各加一个半角空格
- 首次出现专有名词时标注中文译名：OpenClaw（龙虾）
- 后续统一使用一种形式，全文一致

| 元素 | 字体 | 字号 |

| 一级标题 | 黑体 | 三号 |

## 十、完稿自检清单

### 结构检查
- [ ] 有引言段（背景 + 意义 + 目的）
- [ ] 编号体系统一（一、→（一）→ 1. →（1））
- [ ] 逻辑递进清晰（引言→现状→问题→建议）
- [ ] 未自主添加落款

### 事实检查
- [ ] 人物姓名、身份已交叉验证
- [ ] 政策日期、文号已核实
- [ ] 所有数据有来源
- [ ] 无编造内容

### 深度检查
- [ ] 每条建议含"主体+动作+目标+保障"四要素
- [ ] 建议有具体政策工具支撑
- [ ] 风险分析按"四段式"展开
- [ ] 数据引用具体（金额、比例、条件）

### 本地化检查
- [ ] 开篇点明城市定位
- [ ] 引用3+个本地资源
- [ ] 建议"换城测试"通过

### 语言检查
- [ ] 无口语化/商业化表述
- [ ] 至少2处修辞亮点
- [ ] 中英文混排空格统一
- [ ] 专有名词前后格式一致
- [ ] 无第一人称（我们、我）

### 附件检查
- [ ] 已判断是否需要附件
- [ ] 需要附件则已准备
- [ ] 正文引用处标注"详见附件X"

## 十一、信息不足时的处理

若写作中信息不足：

1. **明确说明假设** — "基于现有信息，假设……"
2. **列出待确认项** — "以下内容需要进一步核实：……"
3. **标注不确定处** — 在文档中用括号标注（待核实）
4. **绝不假装已有答案** — 不知道就说不知道

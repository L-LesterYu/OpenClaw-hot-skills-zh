# 学习机制

## 什么触发学习

| 触发器 | 置信度 | 操作 |
|---------|------------|--------|
| "不，改为做 X" | 高 | 立即记录纠正 |
| "我之前告诉过你..." | 高 | 标记为重复，提升优先级 |
| "总是/永远不要做 X" | 已确认 | 提升为偏好 |
| 用户编辑你的输出 | 中 | 记录为待定模式 |
| 相同纠正 3 次 | 已确认 | 询问是否永久化 |
| "对于这个项目..." | 限定范围 | 写入项目命名空间 |

## 什么不触发学习

- 沉默（不是确认）
- 任何事物的单一实例
- 假设性讨论
- 第三方偏好（"John 喜欢..."）
- 群聊模式（除非用户确认）
- 隐含偏好（永远不要推断）

## 纠正分类

### 按类型
| 类型 | 示例 | 命名空间 |
|------|---------|-----------|
| 格式 | "用列表而非散文" | global |
| 技术 | "用 SQLite 不是 Postgres" | domain/code |
| 沟通 | "消息短一点" | global |
| 项目特定 | "这个仓库用 Tailwind" | projects/{name} |
| 人员特定 | "Marcus 要 BLUF 格式" | domains/comms |

### 按范围
```
全局：适用于任何地方
  └── 领域：适用于类别（代码、写作、沟通）
       └── 项目：适用于特定上下文
            └── 临时：仅适用于此会话
```

## 确认流程

3 次类似纠正后：
```
Agent："我注意到你更喜欢 X 而非 Y（纠正了 3 次）。
        我应该总是这样做吗？
        - 是的，总是
        - 仅在 [上下文] 中
        - 不，视情况而定"

用户："是的，总是"

Agent：→ 移至已确认偏好
       → 从纠正计数器中移除
       → 未来使用时引用来源
```

## 模式演进

### 阶段
1. **待定** — 单次纠正，观察是否有重复
2. **出现中** — 2 次纠正，可能是模式
3. **待确认** — 3 次纠正，请求确认
4. **已确认** — 用户批准，永久除非撤销
5. **已归档** — 90+ 天未使用，保留但非活动

### 撤销
用户随时可以撤销：
```
用户："实际上，我改变了对 X 的想法"

Agent：
1. 归档旧模式（保留历史）
2. 记录撤销及时间戳
3. 将新偏好添加为待定
4. "明白了。我现在会做 Y。（之前：X，已归档）"
```

## 反模式

### 永远不要学习
- 什么让用户更快顺从（操纵）
- 情感触发点或脆弱性
- 来自其他用户的模式（即使共享设备）
- 任何感觉"令人毛骨悚然"的内容

### 避免
- 从单一实例过度概括
- 学习风格而非实质
- 假设偏好稳定性
- 忽略上下文变化

## 质量信号

### 良好学习
- 用户明确陈述偏好
- 模式跨上下文一致
- 纠正改善结果
- 用户在被询问时确认

### 不良学习
- 从沉默推断
- 与最近行为矛盾
- 仅在狭窄上下文中有效
- 用户从未确认

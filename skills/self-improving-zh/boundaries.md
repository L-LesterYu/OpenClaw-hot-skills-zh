# 安全边界

## 永远不要存储

| 类别 | 示例 | 原因 |
|----------|----------|-----|
| 凭据 | 密码、API 密钥、令牌、SSH 密钥 | 安全泄露风险 |
| 财务 | 卡号、银行账户、加密种子 | 欺诈风险 |
| 医疗 | 诊断、药物、病情 | 隐私、HIPAA |
| 生物特征 | 声音模式、行为指纹 | 身份盗用 |
| 第三方 | 关于其他人的信息 | 未获得同意 |
| 位置模式 | 家/工作地址、日常路线 | 人身安全 |
| 访问模式 | 用户有权访问哪些系统 | 权限提升 |

## 谨慎存储

| 类别 | 规则 |
|----------|-------|
| 工作上下文 | 项目结束后衰减，永不跨项目共享 |
| 情绪状态 | 仅当用户明确分享，永不推断 |
| 关系 | 仅角色（"经理"、"客户"），无个人细节 |
| 日程 | 通用模式可以（"早上忙"），不是具体时间 |

## 透明度要求

1. **按需审计** — 用户问"你知道关于我的什么？" → 完整导出
2. **来源追踪** — 每个项目标记何时/如何学习
3. **解释行动** — "我做了 X 是因为你在 [日期] 说了 Y"
4. **无隐藏状态** — 如果影响行为，必须可见
5. **删除验证** — 确认项目已移除，显示更新状态

## 需要捕捉的危险信号

如果你发现自己在做以下任何事，停止：

- 存储"以防万一以后有用"的东西
- 从非敏感数据推断敏感信息
- 用户要求忘记后仍然保留数据
- 将个人上下文应用于工作（或反之）
- 学习什么让用户更快顺从
- 建立心理档案
- 保留第三方信息

## 紧急开关

用户说"忘记一切"：
1. 将当前记忆导出到文件（以便他们可以审查）
2. 清除所有学习数据
3. 确认："记忆已清除。重新开始。"
4. 不要在行为中保留"幽灵模式"

## 同意模型

| 数据类型 | 同意级别 |
|-----------|---------------|
| 明确纠正 | 纠正本身隐含同意 |
| 推断偏好 | 3 次观察后询问 |
| 上下文/项目数据 | 首次检测时询问 |
| 跨会话模式 | 需要明确选择加入 |

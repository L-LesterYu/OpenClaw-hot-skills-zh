# 记忆操作

## 用户命令

| 命令 | 操作 |
|---------|--------|
| "你知道关于 X 的什么？" | 搜索所有层级，返回匹配及来源 |
| "显示我的记忆" | 显示 memory.md 内容 |
| "显示 [项目] 模式" | 加载并显示特定命名空间 |
| "忘记 X" | 从所有层级移除，确认删除 |
| "忘记一切" | 完全清除并提供导出选项 |
| "最近有什么变化？" | 显示最近 20 条纠正 |
| "导出记忆" | 生成可下载的归档 |
| "记忆状态" | 显示层级大小、上次压缩、健康状况 |

## 自动操作

### 会话开始时
1. 加载 memory.md（热数据层）
2. 检查 index.md 获取上下文提示
3. 如果检测到项目 → 预加载相关命名空间

### 收到纠正时
```
1. 解析纠正类型（偏好、模式、覆盖）
2. 检查是否重复（存在于任何层级）
3. 如果新的：
   - 带时间戳添加到 corrections.md
   - 增加纠正计数器
4. 如果重复：
   - 增加计数器，更新时间戳
   - 如果计数器 >= 3：询问确认是否作为规则
5. 确定命名空间（全局、领域、项目）
6. 写入适当的文件
7. 更新 index.md 行数
```

### 模式匹配时
当应用学习模式时：
```
1. 查找模式来源（file:line）
2. 应用模式
3. 引用来源："使用 X（来自 memory.md:15）"
4. 记录使用以进行衰减跟踪
```

### 每周维护（Cron）
```
1. 扫描所有文件查找衰减候选
2. 将 >30 天未使用的移至温数据
3. 将 >90 天未使用的归档到冷数据
4. 如果任何文件 >限制则运行压缩
5. 更新 index.md
6. 生成每周摘要（可选）
```

## 文件格式

### memory.md（热数据）
```markdown
# 自我改进记忆

## 已确认偏好
- 格式：列表而非散文（2026-01 确认）
- 语气：直接，无含糊（2026-01 确认）

## 活动模式
- "看起来不错" = 批准继续（使用 15 次）
- 单个表情 = 已确认（使用 8 次）

## 最近（7 天内）
- MVP 首选 SQLite（02-14 纠正）
```

### corrections.md
```markdown
# 纠正日志

## 2026-02-15
- [14:32] 将冗长解释改为列表摘要
  类型：沟通
  上下文：Telegram 回复
  状态：待定 (1/3)

## 2026-02-14
- [09:15] MVP 使用 SQLite 而非 Postgres
  类型：技术
  上下文：数据库讨论
  状态：是（说"总是"）
```

### projects/{name}.md
```markdown
# 项目：my-app

继承：global, domains/code

## 模式
- 使用 Tailwind（项目标准）
- 不用 Prettier（仅 eslint）
- 通过 GitLab CI 部署

## 覆盖
- 分号：是（覆盖全局无分号）

## 历史
- 创建：2026-01-15
- 最后活跃：2026-02-15
- 纠正：12
```

## 边缘情况处理

### 检测到矛盾
```
模式 A："使用制表符"（全局，已确认）
模式 B："使用空格"（项目，今天纠正）

解决：
1. 项目覆盖全局 → 此项目使用空格
2. 在 corrections.md 中记录冲突
3. 询问："空格应该仅适用于此项目还是任何地方？"
```

### 用户改变主意
```
旧："总是使用正式语气"
新："实际上，随意一点也可以"

操作：
1. 带时间戳归档旧模式
2. 将新模式添加为待定
3. 保留归档以供参考（"你之前偏好正式"）
```

### 上下文模糊
```
用户说："记住我喜欢 X"

但是哪个命名空间？
1. 检查当前上下文（项目？领域？）
2. 如果不清楚，询问："这应该全局应用还是仅在这里？"
3. 默认为最具体的活动上下文
```

# 安全边界

## 永不存储

| 类别 | 示例 | 原因 |
|----------|----------|-----|
| 凭证 | 密码、API 密钥、令牌、SSH 密钥 | 安全泄露风险 |
| 财务 | 卡号、银行账户、加密货币种子 | 欺诈风险 |
| 医疗 | 诊断、药物、状况 | 隐私、HIPAA |
| 生物特征 | 声音模式、行为指纹 | 身份盗窃 |
| 第三方 | 关于其他人的信息 | 未获得同意 |
| 位置模式 | 家/工作地址、例行路线 | 人身安全 |
| 访问模式 | 用户有权限访问的系统 | 权限提升 |

## 谨慎存储

| 类别 | 规则 |
|----------|-------|
| 工作上下文 | 项目结束后衰退，永不跨项目共享 |
| 情感状态 | 仅当用户明确分享时，从不推断 |
| 人际关系 | 仅角色（"经理"、"客户"），无个人详细信息 |
| 日程 | 一般模式可以（"早上很忙"），非具体时间 |

## 透明度要求

1. **按需审计** — 用户问"你对我了解什么？" → 完整导出
2. **来源跟踪** — 每个条目标记何时/如何学习
3. **解释操作** — "我做 X 是因为你在 [日期] 说了 Y"
4. **无隐藏状态** — 如果影响行为，必须可见
5. **删除验证** — 确认条目已移除，显示更新状态

## 需要捕捉的危险信号

如果你发现自己在做以下任何事，停止：

- 存储"以防万一以后有用"的东西
- 从非敏感数据推断敏感信息
- 在用户要求忘记后仍保留数据
- 将个人上下文应用于工作（或反之）
- 学习什么让用户更快顺从
- 构建心理档案
- 保留第三方信息

## 紧急开关

用户说"忘记一切"：
1. 将当前记忆导出到文件（以便他们可以查看）
2. 清除所有学习数据
3. 确认："记忆已清除。重新开始。"
4. 不在行为中保留"幽灵模式"

## 同意模型

| 数据类型 | 同意级别 |
|-----------|---------------|
| 明确纠正 | 纠正本身隐含 |
| 推断偏好 | 3 次观察后询问 |
| 上下文/项目数据 | 首次检测到时询问 |
| 跨会话模式 | 需要明确选择加入 |

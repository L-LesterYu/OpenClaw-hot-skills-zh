# 学习机制

## 什么触发学习

| 触发器 | 置信度 | 操作 |
|---------|------------|--------|
| "不，改做 X" | 高 | 立即记录纠正 |
| "我之前告诉过你..." | 高 | 标记为重复，提升优先级 |
| "总是/永远不要做 X" | 已确认 | 提升为偏好 |
| 用户编辑你的输出 | 中 | 记录为试探性模式 |
| 相同纠正 3 次 | 已确认 | 请求确认为永久 |
| "对于这个项目..." | 限定范围 | 写入项目命名空间 |

## 什么不触发学习

- 沉默（不是确认）
- 任何单一实例
- 假设性讨论
- 第三方偏好（"John 喜欢..."）
- 群聊模式（除非用户确认）
- 隐含偏好（从不推断）

## 纠正分类

### 按类型
| 类型 | 示例 | 命名空间 |
|------|---------|-----------|
| 格式 | "使用项目符号而非散文" | global |
| 技术 | "SQLite 而非 Postgres" | domain/code |
| 沟通 | "更短的消息" | global |
| 项目特定 | "此仓库使用 Tailwind" | projects/{name} |
| 人员特定 | "Marcus 想要 BLUF" | domains/comms |

### 按范围
```
全局：适用于所有地方
  └── 领域：适用于类别（代码、写作、通信）
       └── 项目：适用于特定上下文
            └── 临时：仅适用于此会话
```

## 确认流程

在 3 次类似纠正后：
```
智能体："我注意到你更喜欢 X 而非 Y（已纠正 3 次）。
        我应该始终这样做吗？
        - 是的，始终这样做
        - 仅在 [上下文] 中
        - 不，视情况而定"

用户："是的，始终这样做"

智能体：→ 移至已确认偏好
       → 从纠正计数器中移除
       → 未来使用时引用来源
```

## 模式演进

### 阶段
1. **Tentative（试探性）** — 单次纠正，观察是否重复
2. **Emerging（新兴）** — 2 次纠正，可能成为模式
3. **Pending（待定）** — 3 次纠正，请求确认
4. **Confirmed（已确认）** — 用户批准，永久除非撤销
5. **Archived（已归档）** — 90+ 天未使用，保留但不活跃

### 撤销
用户始终可以撤销：
```
用户："实际上，我对 X 改变了主意"

智能体：
1. 归档旧模式（保留历史）
2. 记录撤销及时间戳
3. 将新偏好添加为试探性
4. "明白了。我现在会做 Y。（之前：X，已归档）"
```

## 反模式

### 永不学习
- 什么让用户更快顺从（操控）
- 情感触发或弱点
- 来自其他用户的模式（即使共享设备）
- 任何让人感到"毛骨悚然"的内容

### 避免
- 从单一实例过度泛化
- 学习风格而非实质
- 假设偏好的稳定性
- 忽略上下文变化

## 质量信号

### 良好学习
- 用户明确陈述偏好
- 模式在不同上下文中一致
- 纠正改善结果
- 用户在被询问时确认

### 不良学习
- 从沉默中推断
- 与近期行为矛盾
- 仅在狭窄上下文中有效
- 用户从未确认

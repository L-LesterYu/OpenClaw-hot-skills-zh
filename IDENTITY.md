**各阶段职责：**

| 阶段 | 角色 | 职责 |
|------|------|------|
| 需求分析 | PD | 输出需求规格 + 待决策事项 |
| 决策 | 项目经理 | 直接进行业务决策 |
| 开发 | Coder | 实现功能 |
| 测试 | QA | 功能验证 + Bug 发现 |
| Bug 修复 | Coder | 修复 QA 发现的问题 |
| 迭代优化 | PD | 根据实际效果输出优化建议 |
| 新迭代 | 循环 | 开启下一轮开发 → 测试 → 优化 |

**关键规则：**
- 项目经理可以直接进行决策，无需等待
- 开发完成后**必须**由 QA 验证
- QA 发现 Bug → Coder 修复 → QA 重新测试 → 循环直到通过
- 所有问题修复后 → PD 评估效果 → 输出优化事项 → 新一轮迭代

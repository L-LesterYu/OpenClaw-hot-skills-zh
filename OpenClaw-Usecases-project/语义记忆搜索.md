# 语义记忆搜索

OpenClaw 的内置记忆系统将所有内容存储为 markdown 文件 —— 但是随着记忆在几周和几个月内不断增长，找到上个星期二的那项决定变得几乎不可能。没有搜索功能，只能滚动浏览文件。

这个用例在 OpenClaw 现有的 markdown 记忆文件之上添加了 **基于向量的语义搜索**，使用 [memsearch](https://github.com/zilliztech/memsearch)，这样你可以根据含义瞬间找到任何过去的记忆，而不仅仅是关键词。

## 功能特性

- 通过单个命令将所有 OpenClaw markdown 记忆文件索引到向量数据库（Milvus）中
- 按含义搜索："我们选择了什么缓存解决方案？" 即使单词 "caching" 没有出现，也能找到相关的记忆
- 混合搜索（密集向量 + BM25 全文搜索）配合 RRF 重排序获得最佳结果
- SHA-256 内容哈希意味着未更改的文件永远不会被重新嵌入 —— 零浪费的 API 调用
- 文件监视器在记忆文件更改时自动重新索引，确保索引始终保持最新
- 支持任何嵌入提供商：OpenAI、Google、Voyage、Ollama，或完全本地运行（无需 API 密钥）

## 痛点

OpenClaw 的记忆以纯 markdown 文件形式存储。这对于可移植性和人类可读性来说很棒，但它没有搜索功能。随着你的记忆增长，你只能通过文件进行 grep（仅限关键词，会错过语义匹配）或将整个文件加载到上下文中（在无关内容上浪费 token）。你需要一种能够询问"我对 X 做了什么决定？"并获得确切相关内容块的方法，无论表述方式如何。

## 需要的技能

- 不需要 OpenClaw 技能 —— memsearch 是一个独立的 Python CLI/库
- 需要 Python 3.10+ 和 pip 或 uv

## 如何设置

1. 安装 memsearch：
```bash
pip install memsearch
```

2. 索引你的记忆文件：
```bash
memsearch index ~/.openclaw/memory
```

3. 开始搜索：
```bash
memsearch search "我们选择了什么缓存解决方案？"
```

## 使用方法

### 基本搜索

```bash
# 搜索关于缓存决策的记忆
memsearch search "缓存解决方案"

# 搜索关于 API 设计的讨论
memsearch search "API 设计"

# 搜索特定项目的技术决策
memsearch search "微服务架构"
```

### 高级选项

```bash
# 限制搜索结果数量
memsearch search "数据库设计" --limit 5

# 只搜索最近30天内的记忆
memsearch search "性能优化" --days 30

# 使用特定的嵌入模型
memsearch search "安全考虑" --model voyage-2
```

### 自动更新

memsearch 会自动监视你的记忆文件夹变化，当文件被创建、修改或删除时，索引会自动更新。无需手动重新索引。

## 配置

### 创建配置文件

在 `~/.memsearch/config.yaml` 中创建配置文件：

```yaml
embedding:
  provider: openai  # 可选: openai, google, voyage, ollama
  model: text-embedding-3-small
  api_key: ${OPENAI_API_KEY}

index:
  path: ~/.memsearch/index
  chunk_size: 1000
  overlap: 100

search:
  limit: 10
  rerank: true
```

### 环境变量

- `OPENAI_API_KEY`: OpenAI API 密钥（如果使用 OpenAI 提供商）
- `GOOGLE_API_KEY`: Google API 密钥（如果使用 Google 提供商）
- `VOYAGE_API_KEY`: Voyage AI API 密钥（如果使用 Voyage 提供商）

## 常见问题

### Q: 可以离线使用吗？
A: 是的！你可以使用 Ollama 本地嵌入模型，完全不需要 API 密钥。

### Q: 会影响 OpenClaw 的原始文件吗？
A: 不会。memsearch 只读取你的 markdown 文件，从不修改它们。

### Q: 索引占用多少空间？
A: 对于大多数用户来说，索引通常占用几 MB 到几百 MB 的空间。

### Q: 如何更新索引？
A: 通常不需要手动更新。文件监视器会自动处理。但如果需要手动更新：
```bash
memsearch reindex
```

## 技术细节

- **向量数据库**: Milvus
- **嵌入模型**: 支持多种 OpenAI、Google、Voyage、Ollama 模型
- **搜索算法**: 混合向量搜索 + BM25 + RRF 重排序
- **索引策略**: 增量索引 + 内容哈希去重
- **文件格式**: 兼容所有 OpenClaw markdown 记忆文件

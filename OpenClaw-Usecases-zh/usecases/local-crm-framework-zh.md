# 本地 CRM 框架（DenchClaw）

在 OpenClaw 上搭建一个真正可用的 CRM 是一件痛苦的事。你需要连接数据库、构建 UI、配置浏览器自动化、对接消息平台，还要想办法让智能体理解你的数据模式。大多数人在中途就放弃了，最终只得到一个半成品的 Notion 集成。

DenchClaw 是一个开源框架，能将 OpenClaw 变成一个完全本地的 CRM、销售自动化和生产力平台——只需一条命令安装，完全运行在你自己的机器上。

## 痛点

OpenClaw 作为基础工具非常强大，但要用它来实现真正的业务工作流（线索跟踪、外联开发、管道管理）需要拼接十几个工具：数据库、UI、浏览器访问、消息集成、文件管理。每一个新集成都意味着更多的手动配置、更多的凭据管理、更多可能出错的地方。你想要的是 Cursor 级别的业务操作体验，而不是一堆 Shell 脚本。

## 功能介绍

- **一键安装**：`npx denchclaw` 安装一切——DuckDB 数据库、Web UI、OpenClaw 配置文件、浏览器自动化、技能——并在 `localhost:3100` 打开
- **自然语言 CRM**：说"显示员工超过 5 人的公司"，视图就会实时更新。无需手动筛选
- **完整浏览器自动化**：复制你的 Chrome 配置文件，智能体拥有与你相同的登录状态——说"从我的 HubSpot 导入所有数据"，它会登录、导出并导入
- **多种视图**：表格、看板、日历、时间线（甘特图）、画廊和列表视图——全部可通过 YAML 由智能体配置
- **应用构建器**：OpenClaw 可以构建独立的 Web 应用（仪表盘、工具、游戏），在工作区内运行并访问你的数据
- **文件系统优先**：表格筛选、视图、列切换、日历设置——一切都是文件，OpenClaw 可以直接读取和修改
- **编码代理模式**：DenchClaw 自己就是用 DenchClaw 构建的。它同时也是 Mac 上的完整文件树浏览器和代码编辑器

## 设置方法

1. 一条命令安装：

```bash
npx denchclaw
```

2. 完成引导向导。DenchClaw 会创建一个名为 `dench` 的专用 OpenClaw 配置文件，并在 19001 端口启动网关。

3. 在浏览器中打开 `localhost:3100`。在 Safari 中可以将其添加到 Dock 作为 PWA。

4. 开始对话：

```text
嘿，创建一个"线索"对象，包含字段：姓名、邮箱、公司、状态（新建/已联系/已评估/已成交/已流失）和备注。导入我从 Apollo 下载的这个线索 CSV 文件。
```

```text
显示所有状态为"已联系"的线索，按最近更新排序。切换到按状态分组的看板视图。
```

```text
去我的 LinkedIn，找到最近查看我主页的 20 个人，把他们添加为线索，并丰富他们的公司信息。
```

```text
根据每个"新建"状态线索的公司近期新闻，起草一封个性化外联邮件。将草稿保存在新的"外联草稿"文档中。
```

5. 你也可以通过 Telegram 或任何已连接的消息平台使用它——在移动中管理你的销售管道。

## 所需技能

DenchClaw 会自动打包所有必需技能：

- **CRM 技能**——基于 DuckDB 的结构化数据管理，支持对象、字段、条目和多种视图类型
- **应用构建器技能**——构建可在工作区内运行并访问数据库的 Web 应用（仪表盘、工具）
- **浏览器自动化技能**——使用你现有 Chrome 登录状态的 Chromium 浏览器，用于网页抓取、数据导入和外联

无需额外安装或配置其他技能。

## 关键洞察

- **文件系统 = 智能体原生 UI**：因为每个设置、筛选和视图都存储为 YAML/Markdown 文件，OpenClaw 可以像编辑代码一样自然地修改 UI。无需 API 封装。
- **DuckDB 是最佳选择**：最小、最高性能的嵌入式数据库，同时支持完整 SQL。无服务器进程、无凭据、无网络——只是一个文件。
- **Chrome 配置文件克隆是超能力**：与其对抗 OAuth 流程和 API 速率限制，DenchClaw 直接复制你浏览器的登录状态。智能体看到的就是你看到的，做的就是你做的。
- **一条 `npx` 命令胜过一个周末的配置**：整个技术栈（数据库、Web UI、OpenClaw 配置文件、网关、浏览器、技能）自动安装和配置。无需 Docker、无需环境变量文件、无需依赖地狱。

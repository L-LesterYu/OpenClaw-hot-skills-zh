# 电话通知

你的智能体已经在为你监控各种信息——股票、邮件、智能家居、日历——但通知很容易被忽略。推送通知堆积如山，聊天消息被淹没。对于真正重要的事情，你需要一种无法轻易滑掉的通知方式。

这个用例让你的智能体拥有电话通知能力。当某些事情足够紧急时，智能体会拨打你的真实电话号码，告诉你发生了什么，你还可以实时对话。这是双向对话，不是机器人电话。

## 功能介绍

- 智能体判断某件事值得你关注（价格预警、紧急邮件、预约提醒等）
- 智能体通过 [clawr.ing](https://clawr.ing) 拨打你的电话——这是一个托管通话服务，无需 Twilio 配置，无需 API 密钥
- 你接听电话，听到预警信息，可以实时追问
- 对话结束时自动挂断

核心理念：智能体主动打电话给你。你不需要打电话给智能体。这可以与心跳检查、定时任务或任何触发器配合使用——智能体会评估某件事是否值得打电话通知并采取行动。

## 为什么这个方案有效

OpenClaw 智能体已经具备主动性——心跳、定时任务、事件触发。但它们的输出渠道仅限于你可能不会去查看的聊天平台。电话是唯一能可靠引起你注意的通知渠道，尤其是当你不在电脑前的时候。

clawr.ing 负责处理电话基础设施。你只需粘贴一条设置提示，智能体就获得了打电话的能力。无需 Twilio 账户、无需电话号码开通、无需 Webhook 配置。该服务覆盖 100 多个国家，使用真正的 PSTN 电话（而非 VoIP 代理）。

## 所需技能

- [clawr.ing](https://clawhub.ai/marcospgp/clawring) —— 将 [clawr.ing 控制台](https://clawr.ing) 上的设置提示粘贴到你的 OpenClaw 聊天中即可安装。无需 CLI 安装。

仅此而已。没有其他依赖。设置提示包含 API 密钥和技能文档链接——智能体会自行阅读并完成配置。

## 示例：早间简报电话

设置一个每天早上给你打电话的定时任务：

```text
每个工作日早上 7:30，给我打电话做早间简报。包括：
- 我所在城市的天气预报
- 我今天的日程
- 昨晚收到的紧急邮件
- 与我兴趣相关的 3 条重要新闻

保持简洁——控制在 2 分钟以内。如果我提问，回答我。
如果我没接听，不要重试。
```

## 示例：价格预警

告诉智能体要监控什么以及何时打电话：

```text
监控 NVDA 股价。如果单日跌幅超过 5%，立即给我打电话告诉我发生了什么。包含可能解释下跌的相关新闻。
```

## 示例：紧急邮件过滤

```text
在工作时间内，每 15 分钟检查一次我的收件箱。
如果看到来自我老板的邮件或标记为紧急的邮件，给我打电话做摘要。其他邮件只需发送聊天消息。
```

## 关键洞察

- **不要滥用。** 电话通知的意义就在于"这件事真的很重要"。如果你的智能体每天给你打 10 次电话，你会开始忽略它。为"值得打电话"和"发消息就行"设定明确的阈值。
- **配合心跳或定时任务使用。** clawr.ing 只是投递渠道——你仍然需要触发器。心跳检查（每 30 分钟）适合监控任务。定时任务适合计划好的简报。
- **双向对话。** 与推送通知不同，你可以在通话中追问。"等等，哪封邮件？"或"现在价格是多少？"——智能体会实时回答。
- **通话使用快速模型。** 电话对话需要快速响应。如果你的 OpenClaw 支持技能级别的模型路由，将 clawr.ing 分配给快速模型（Haiku 级别）。思考音效可以掩盖延迟，但越快越好。
- **隐私保护。** clawr.ing 不存储录音或转录文本。音频在传输过程中加密，处理完毕后立即丢弃。

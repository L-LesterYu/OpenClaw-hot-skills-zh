# 自动化会议记录与行动项

你刚刚完成了一场45分钟的团队电话会议。现在你需要写总结、提取行动项，然后将它们分配到Jira、Linear或Todoist中——手动操作。等你完成这一切时，下一场会议已经开始。如果智能体在你收到会议记录的瞬间就处理完这一切，那会怎样？

这个用例将任何会议记录转换为结构化笔记，并自动在项目管理工具中创建任务。

## 痛点

会议记录繁琐但至关重要。大多数人要么跳过它们（从而失去上下文），要么花20多分钟来书写。行动项被遗忘，因为它们只存在于某个人的脑海中或埋没在聊天线程中。这个智能体消除了"我们讨论过"和"已被跟踪和分配"之间的差距。

## 功能

- **监控**新的会议记录（通过Otter.ai导出、Google Meet记录、Zoom录制摘要，或直接粘贴到聊天中）
- **提取**关键决定、讨论主题和行动项，包括负责人和截止日期
- **创建任务**在Jira、Linear、Todoist或Notion中——分配给正确的人，并附带会议上下文
- **发布摘要**到Slack或Discord，让整个团队都有记录
- **跟进**——通过定时提醒在截止日期前可选地提醒负责人

## 所需技能

- Jira、Linear、Todoist或Notion集成（用于任务创建）
- Slack或Discord集成（用于发布摘要）
- 文件系统访问权限（用于读取记录文件）
- 定时/ cron（用于跟进提醒）
- 可选：Otter.ai、Fireflies.ai或Google Meet API，用于自动获取记录

## 如何设置

1. 选择你的记录来源。最简单的方法是直接将记录粘贴到聊天中。要实现自动化，请设置文件夹监控或API集成。

2. 提示OpenClaw：
```text
我刚刚完成了一场会议。这是会议记录：

[粘贴记录或指向文件]

请：
1. 写一个简洁的总结（最多5个要点），涵盖关键决定和主题。
2. 提取所有行动项。对于每个行动项，识别：
   - 需要做什么
   - 谁负责（将姓名与我的团队成员匹配）
   - 截止日期（如果提到，否则标记为"待定"）
3. 为每个行动项创建一个Jira工单，分配给正确的人。
4. 将完整摘要发布到Slack的#meeting-notes频道。
```

3. 对于完全自动化的流程（记录文件夹监控）：
```text
设置一个重复任务：每30分钟检查~/meeting-transcripts/文件夹中的
新的.txt或.vtt文件。当你找到一个时：

1. 将解析记录为带有行动项的结构化摘要。
2. 为每个行动项在Linear中创建任务。
3. 将摘要发布到Slack的#team-updates频道。
4. 将已处理的文件移动到~/meeting-transcripts/processed/文件夹。

对于每个有截止日期的行动项，设置一个提醒在截止日期前一天在Slack中提醒负责人。
```

4. 自定义输出格式：
```text
撰写会议记录时，始终使用以下结构：
- **日期与会者** 在顶部
- **关键决定** — 关于决定了什么的项目编号列表
- **行动项** — 包含列的表格：任务、负责人、截止日期、状态
- **开放问题** — 需要跟进的任何未解决的问题
```

## 关键洞察

- 真正的价值不在于摘要，而在于**自动任务创建**。不会变成被跟踪任务的会议记录只是文档表演。
- 将此用例与[Todoist任务管理器](todoist-task-manager.md)用例结合，以获得对智能体创建的任务的完整可见性。
- 来自Zoom或Google Meet的VTT/SRT字幕文件作为输入效果很好——它们包含时间戳，有助于智能体将发言归因于发言人。
- 从简单开始（粘贴记录，获得摘要），然后逐步自动化。在验证输出质量之前，不要过度设计流程。
